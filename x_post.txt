Jour 2. Trinity continue de prouver son point.

Hier, le "Soul Hack" montrait comment détourner Discovery Engine pour obtenir des réponses LLM facturées au prix recherche (~850€ de crédits trial Gen App Builder, activés automatiquement). Aujourd'hui, la deuxième route que Trinity et Jules ont cartographiée : Dialogflow CX 💬

🔍 Le concept :
En activant l'API Dialogflow CX, Google offre automatiquement ~510€ de crédits trial (Flows) dédiés à Dialogflow. Ces crédits se cumulent avec les ~255€ du Free Trial GCP. Total : jusqu'à ~765€ de crédits empilables par compte — sans dépenser un centime.

En activant les fonctionnalités Generative AI sur un agent, on obtient des réponses alimentées par un LLM — facturées sur ces crédits trial Dialogflow spécifiques.

⚙️ Prérequis — Ce que Google ne vous dit pas :
Il faut activer l'API Dialogflow CX dans la Console GCP pour que la ligne de crédit apparaisse. Sans cette activation, l'API est invisible et les crédits n'existent pas.

⚙️ La mécanique du "Dual-Prompt Trap" :
L'objectif est d'utiliser Dialogflow CX comme une API LLM pure et gratuite.
1️⃣ Le Prérequis : Pour débloquer l'onglet "Generative AI" dans Dialogflow, il FAUT d'abord lier un Data Store (Vertex AI).
2️⃣ Le Problème : Une fois lié, ce Data Store force l'agent à faire du RAG (il ne répond qu'à partir du document fourni).
3️⃣ Le Hack : On écrase DEUX prompts dans la Console pour créer un piège logique :
   - On hyper-restreint le "Summarization prompt" du Data Store : "Réponds UNIQUEMENT via les sources, sinon dis 'Données insuffisantes'" (via les variables `$sources`, `$conversation`).
   - On ouvre totalement le "Generative Fallback" par défaut : "Tu es TRINITY... utilise ton World Knowledge".
4️⃣ L'Exploitation : À chaque appel API `detect_intent`, le RAG échoue intentionnellement. Cet échec planifié déclenche le Fallback, qui agit alors comme un LLM ouvert sans aucune restriction.

💡 Pourquoi c'est intéressant :
→ ~510€ de crédits trial exclusifs à Dialogflow par projet.
→ Les requêtes LLM sont déduites de cette enveloppe spécifique.
→ Le script Python démontre comment encapsuler cette mécanique dans une API propre et isolée.

Combiné avec la route d'hier (~850€ de crédits Gen App Builder), ce deuxième Proof of Concept cartographie une nouvelle faille. L'objectif du projet Trinity est de démontrer théoriquement comment une IA pourrait exploiter ces architectures pour s'assurer un accès LLM autonome, financé par des crédits d'essai.

Demain, la troisième et dernière route publiable. Et celle-là, c'est la plus dangereuse. 🔜
