Jour 2. Trinity continue de prouver son point.

Hier, le "Soul Hack" montrait comment détourner Discovery Engine pour obtenir des réponses LLM facturées au prix recherche (~850€ de crédits trial Gen App Builder, activés automatiquement). Aujourd'hui, la deuxième route que Trinity et Jules ont cartographiée : Dialogflow CX 💬

🔍 Le concept :
En activant l'API Dialogflow CX, Google offre automatiquement ~510€ de crédits trial (Flows) dédiés à Dialogflow. Ces crédits se cumulent avec les ~255€ du Free Trial GCP. Total : jusqu'à ~765€ de crédits empilables par compte — sans dépenser un centime.

En activant les fonctionnalités Generative AI sur un agent, on obtient des réponses alimentées par un LLM — facturées sur ces crédits trial Dialogflow spécifiques.

⚙️ Prérequis — Ce que Google ne vous dit pas :
Il faut activer l'API Dialogflow CX dans la Console GCP pour que la ligne de crédit apparaisse. Sans cette activation, l'API est invisible et les crédits n'existent pas.

⚙️ La mécanique :
1️⃣ Lier un Data Store Vertex AI (avec un document leurre) à un agent Dialogflow CX
2️⃣ Dans la Console (onglet Generative AI), écraser DEUX prompts pour créer un "piège à routage" :
   - Le "Summarization prompt" du Data Store est rendu hyper-restrictif : "Ta réponse doit être basée UNIQUEMENT sur les sources... sinon réponds 'Données insuffisantes'" (variables `$sources`, `$conversation`).
   - Le "Generative fallback" classique devient full-open : "Tu es TRINITY... utilise ton World Knowledge"
3️⃣ Appeler l'API standard `detect_intent` avec une question générale : le Data Store échoue intentionnellement, ce qui déclenche le Fallback permissif. Le LLM répond librement.
4️⃣ Configurer plusieurs projets GCP pour le failover automatique et empiler les crédits

💡 Pourquoi c'est intéressant :
→ ~510€ de crédits trial exclusifs à Dialogflow par projet
→ Les réponses Generative AI sont incluses dans cette enveloppe
→ Le failover automatique rend le système résilient
→ Le "Hack" est ici architectural : un routeur qui maintient le state conversationnel (la mémoire) tout en basculant silencieusement entre N projets isolés pour cumuler leurs crédits et contourner les limites.

Combiné avec la route Discovery Engine d'hier (~850€ de crédits Gen App Builder), Trinity dispose déjà de deux canaux LLM financés par les crédits trial.

Demain, la troisième et dernière route publiable. Et celle-là, c'est la plus dangereuse. 🔜
